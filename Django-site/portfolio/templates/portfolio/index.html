Setting Up Virtual Environment

To start a project, we create a virtual environment to work locally. Bellow we will create this venv virtual environment and name it env.

python -m venv env

This will create a folder for us called env which contains the virtual environment. It contains the files needed, like the Python interpreter and the pip library.

We then need to activate the virtual environment by using source and directing it to the path of the activate script file within our env folder. To do that we do the following command in the terminal:

source env/bin/activate

Doing this uses source command which is used to activate the virtual environment. The activate script will adjusts environment variables to the virtual environment's directories so that Python runs from this isolated environment. At this point, you should see that you are now in your virtual environment.

To deactivate the virtual environment and return to your global Python setup, you can use the deactivate command.

deactivate

Creating A DJango Project

The next step would be to install Django to our virtual environment. We do this with the following command: 

# Latest Version
$ pip install django

# Or a specific version like below:
$ pip install django==3.6

and then we will create a django project with the following command:

django-admin startproject myprojectname

This file will contain all the configuration files needed to run the project. In my case I created the project with this command: 

# The dot at the end is to create a file in the current directory
$ django-admin startproject config .

You should now see we have a config folder and a manage.py file in your project structure.

Now with this set up we can create an app in our Django project. Apps are meant to serve as one function or intent in a Django project. For this project, we are setting up a Rest API for a recipe app. So we will create the app as shown bellow. 

# recipes can be what ever you want to name your app
$  python manage.py startapp recipes

and you should see that app folder created in your folder structure.

To ensure you can use the app in your project, navigate to your project folder and add the app name to your installed apps. In my case it will be config > settings.py and there will be a list (array) of installed apps. Below is an example where I added games to that list of apps and you will need to do this with every app you create. 

    # Application definition

    INSTALLED_APPS = [
        'django.contrib.admin',
        'django.contrib.auth',
        'django.contrib.contenttypes',
        'django.contrib.sessions',
        'django.contrib.messages',
        'django.contrib.staticfiles',
        'recipes',
    ]
